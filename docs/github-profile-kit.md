## Pinned repositories

Pin `Doze52` first.

Then complete the board with repos that reinforce product taste, not just technical breadth.

Recommended order:

1. `Doze52`
2. A repo that shows interface or interaction quality
3. A repo that shows product systems, ops, process, or workflow design
4. A case, experiment, or prototype that shows how you test ideas
5. A practical utility repo only if it reinforces your product lens
6. Another repo with strong narrative value, or leave the sixth slot empty until you have one

Rule:

- If a repo needs a long explanation to look valuable, do not pin it.
- If a repo looks like noise, archive it or keep it unpinned.
- A tighter board with 4 strong pins is better than 6 random ones.

## Repo descriptions

Use this formula:

`What it solves + who it is for + current stage`

Recommended description for `Doze52`:

`Visual year planning tool for focus, routines, and long-range organization. Early public product built with Next.js and Supabase.`

Alternative:

`Year-at-a-glance planning product for people who think in months, not only tasks. Early public build in active evolution.`

Template for other repos:

`[What it solves] for [who it is for]. [Stage or angle].`

Examples:

- `Interaction explorations for product ideas before they become features. Working experiments and UX spikes.`
- `Product operations and workflow experiments for teams that need clarity without heavy process.`
- `Small tools and prototypes used to test product decisions quickly.`
